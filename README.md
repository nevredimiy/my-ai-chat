# My AI Chat Plugin

Плагин `my-ai-chat` добавляет на сайт WordPress простой чат-бот, который отвечает на вопросы по контенту сайта и товарам WooCommerce.

## Поддержка локальной среды

Плагин работает в двух основных сценариях:

- `nginx` / Apache на хост-машине
- WordPress внутри Docker-контейнера

### Как это работает

Плагин добавляет REST API-эндпоинт:

- `/index.php?rest_route=/aibot/v1/chat`

JS-фронтенд отправляет запрос на этот URL, а серверный PHP код обращается к Ollama.

## Настройка Ollama

Плагин автоматически пытается найти работающий Ollama по следующим адресам:

1. `http://host.docker.internal:11434` — удобно для WordPress в Docker, когда Ollama работает на хосте
2. `http://127.0.0.1:11434` — если WordPress и Ollama запущены на одной машине без Docker
3. `http://localhost:11434` — стандартный локальный адрес

Если ни один из адресов не работает, используйте явную настройку через константу или переменную окружения.

### Настройка через константу

Добавьте в `wp-config.php`:

```php
// URL Ollama без слэша в конце
define( 'MY_AI_CHAT_OLLAMA_URL', 'http://host.docker.internal:11434' );

// Имя модели Ollama
define( 'MY_AI_CHAT_OLLAMA_MODEL', 'qwen2.5:1.5b' );

// Если нужна авторизация Ollama
define( 'MY_AI_CHAT_OLLAMA_API_KEY', 'ваш_ключ' );
```

### Настройка через переменные окружения

Можно задавать те же параметры через переменные окружения:

- `MY_AI_CHAT_OLLAMA_URL`
- `MY_AI_CHAT_OLLAMA_MODEL`
- `MY_AI_CHAT_OLLAMA_API_KEY`

## Зависимость от модели

Плагин по умолчанию использует модель:

- `qwen2.5:1.5b`

Если в вашей Ollama установлена другая модель, укажите её через `MY_AI_CHAT_OLLAMA_MODEL`.

## Примеры для разных серверов

### 1. nginx / локальный WordPress без Docker

Ollama должен быть доступен по `http://127.0.0.1:11434` или `http://localhost:11434`.

Если Ollama запускается локально, обычно достаточно настроить:

```php
define( 'MY_AI_CHAT_OLLAMA_URL', 'http://127.0.0.1:11434' );
```

### 2. WordPress в Docker

Если WordPress работает в контейнере, `localhost` внутри контейнера не является хостом машины.

Для доступа к Ollama на хосте используйте:

```php
define( 'MY_AI_CHAT_OLLAMA_URL', 'http://host.docker.internal:11434' );
```

### 3. Ollama и WordPress в одной сети Docker

Если Ollama запущена в другом контейнере и доступна по сервису Docker, укажите внутренний адрес контейнера:

```php
define( 'MY_AI_CHAT_OLLAMA_URL', 'http://ollama:11434' );
```

(тогда Ollama должен быть в той же сети Docker и доступен по имени `ollama`).

## Проверка работоспособности

1. Убедитесь, что WordPress плагин активен.
2. Перейдите на сайт и откройте чат-виджет.
3. Если появилось сообщение `Ошибка соединения с сервером`, проверьте:
   - доступность `Ollama` через указанный URL
   - что REST-эндпоинт WordPress доступен

## Частые ошибки

- `404 Not Found` для `/wp-json/aibot/v1/chat`
  - чаще всего это значит, что REST API не доступен по красивым ссылкам в вашей конфигурации. В этом случае запрос работает через: `/index.php?rest_route=/aibot/v1/chat`.

- `Ошибка соединения с сервером` в чате
  - Ollama недоступна по выбранному адресу
  - модель не установлена в Ollama
  - если нужно, укажите `MY_AI_CHAT_OLLAMA_URL` явно

## Резюме

Плагин должен работать в любой локальной среде, если правильно настроена ссылка на Ollama и рабочая модель.

- `Docker` → `host.docker.internal`
- `nginx / Apache` → `127.0.0.1` или `localhost`
- модель по умолчанию — `qwen2.5:1.5b`
