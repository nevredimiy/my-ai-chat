# My AI Chat (RAG Bot for WordPress & WooCommerce)

Интеллектуальный плагин чат-ассистента для WordPress и WooCommerce, реализующий полноценную архитектуру **RAG (Retrieval-Augmented Generation)** на полностью локальной инфраструктуре. 

Плагин векторизует контент сайта (товары, статьи, страницы), сохраняет их в векторную базу данных и использует LLM для генерации умных ответов на основе контекста магазина с выводом реальных ссылок на товары.

---

## 🏗️ Архитектура системы

Вся цепочка работает локально и не зависит от сторонних платных API (OpenAI, Anthropic и др.):
1. **WordPress + WooCommerce** — источник данных (контент, цены, артикулы, постоянные ссылки).
2. **Ollama (`nomic-embed-text`)** — мультиязычная эмбеддинг-модель, превращающая текст товаров/записей в 768-мерные векторы.
3. **Qdrant (Docker)** — высокопроизводительная векторная база данных для хранения точек (векторов) и их Payload (метаданных). Коллекция `wp_products_collection` создается автоматически при активации плагина.
4. **Ollama (`qwen2.5:1.5b`)** — легковесная и быстрая локальная LLM для генерации финального ответа пользователю на основе контекста, извлеченного из базы данных.

---

## 🛠️ Стек и окружение разработки

* **ОС:** Linux (Ubuntu / Lubuntu)
* **Среда:** Docker Compose (Стек: WordPress PHP, MySQL, phpMyAdmin)
* **Векторная БД:** Qdrant в отдельном Docker-контейнере (`qdrant/qdrant`)
* **ИИ-движок:** Локальный сервер Ollama

---

## 🚀 Настройка и запуск локального окружения

### 1. Запуск Qdrant
Запустите Qdrant в Docker-контейнере на локальном компьютере:
```bash
docker run -d --name qdrant-local \
  -p 6333:6333 -p 6334:6334 \
  -v qdrant_storage:/qdrant/storage \
  qdrant/qdrant
```

### 2. Подготовка моделей в Ollama
Убедитесь, что на локальном сервере Ollama скачаны нужные модели:
```bash
# Модель для генерации векторных представлений (эмбеддингов)
ollama pull nomic-embed-text

# Основная языковая модель для генерации ответов
ollama pull qwen2.5:1.5b
```

### 3. Активация плагина
При активации плагина в админ-панели WordPress автоматически отправляется PUT-запрос в Qdrant для создания коллекции `wp_products_collection` с размерностью 768 векторов и косинусной метрикой расстояния.

## ⚙️ Конфигурация параметров

По умолчанию плагин пытается автоматически найти запущенный Ollama по стандартным адресам, но вы можете переопределить их.

### Настройка через константы в `wp-config.php`
Добавьте в ваш `wp-config.php`:
```php
// URL Ollama (без слэша на конце)
define( 'MY_AI_CHAT_OLLAMA_URL', 'http://host.docker.internal:11434' );

// Имя модели Ollama для генератора
define( 'MY_AI_CHAT_OLLAMA_MODEL', 'qwen2.5:1.5b' );

// Если требуется авторизация к Ollama
define( 'MY_AI_CHAT_OLLAMA_API_KEY', 'ваш_ключ_авторизации' );
```

### Настройка через переменные окружения
Те же параметры можно передать в контейнер WordPress через переменные окружения:
- `MY_AI_CHAT_OLLAMA_URL`
- `MY_AI_CHAT_OLLAMA_MODEL`
- `MY_AI_CHAT_OLLAMA_API_KEY`

Адрес Qdrant жестко настроен в плагине как `http://host.docker.internal:6333`. Если вы запускаете среду вне Docker, убедитесь, что хосты настроены корректно в файле плагина `my-ai-rag-bot.php`.

## 🔄 Синхронизация и индексация данных

### 1. Автоматическая синхронизация при сохранении
Плагин подключается к хуку `save_post`. Каждый раз, когда вы создаете или обновляете товар WooCommerce, страницу или запись блога:
- Содержимое очищается от HTML-тегов.
- Для товаров извлекаются артикул (SKU) и цена.
- Формируется текстовое описание, отправляемое в Ollama (`nomic-embed-text`).
- Полученный вектор сохраняется в Qdrant с ID поста в качестве идентификатора точки.

### 2. Массовая индексация (WP-CLI)
Для первичной загрузки всех существующих товаров, страниц и постов в Qdrant используйте встроенную команду WP-CLI:
```bash
# Если WordPress работает локально
wp ai-bot index --batch_size=50

# Если WordPress работает в Docker-контейнере
docker exec -it wp_php_wp_68 php wp-cli.phar ai-bot index --batch_size=50 --allow-root
```
*Параметр `--batch_size` определяет количество записей, обрабатываемых за одну итерацию.*

## 🧪 Тестирование через WP-CLI

Вы можете проверять работу RAG-цепочки и семантического поиска прямо в терминале:

### 1. Проверка семантического поиска в Qdrant
Ищет топ-3 релевантных объекта по векторному сходству:
```bash
docker exec -it wp_php_wp_68 php wp-cli.phar ai-bot search "червона кепка" --allow-root
```
**Пример вывода:**
```
Ищем: червона кепка
- [ID 123] Кепка червона (Схожесть: 0.8124)
- [ID 124] Футболка червона (Схожесть: 0.6541)
```

### 2. Полноценный тест RAG (Вопрос + Поиск + LLM генерация)
Генерирует финальный ответ со ссылкой на товар, используя контекст из Qdrant:
```bash
docker exec -it wp_php_wp_68 php wp-cli.phar ai-bot ask "Я шукаю червону кепку, у вас є?" --allow-root
```
**Пример вывода:**
```
Вопрос боту: Я шукаю червону кепку, у вас є?
Думаю...

================ ОТВЕТ БОТА ================
Дуже гарна кепка червоного кольору. Розмір 56. (http://localhost:8081/?product=%d0%ba%d0%b5%d0%bf%d0%ba%d0%b0-%d1%87%d0%b5%d1%80%d0%b2%d0%be%d0%bd%d0%b0).
============================================
```

## 💬 Виджет чата на сайте (Web Chat)

Плагин выводит интерактивное окно чата в футере сайта и регистрирует REST API эндпоинт:
- `/index.php?rest_route=/aibot/v1/chat` (или `/wp-json/aibot/v1/chat` при настроенных ЧПУ).

### Особенности работы виджета:
* **Потоковая передача ответов (SSE / Server-Sent Events):** Ответ от Ollama генерируется и выводится в чат в режиме реального времени (по токенам), что повышает отзывчивость интерфейса.
* **Поиск контекста:** Для веб-чата поиск контекста в текущей версии реализован на базе оптимизированного стандартного поиска `WP_Query` по базе данных WordPress, после чего контекст передается модели Ollama для генерации.

## 🛠️ Частые ошибки и их решение

* **`Ошибка соединения с сервером` в чате / Ошибка Ollama API:**
  - Проверьте, запущен ли сервер Ollama.
  - Убедитесь, что модель `qwen2.5:1.5b` загружена (`ollama list`).
  - Проверьте доступность хоста `host.docker.internal` изнутри контейнера WordPress.

* **Векторы не сохраняются в Qdrant:**
  - Убедитесь, что контейнер `qdrant-local` запущен и доступен по порту `6333`.
  - Убедитесь, что в Ollama скачана модель эмбеддингов `nomic-embed-text`.
  - Проверьте логи ошибок PHP (`wp-content/debug.log`).

* **Ошибка `404 Not Found` при обращении к `/wp-json/...`:**
  - На вашем сервере могут быть отключены ЧПУ (Permalinks). Используйте альтернативный роут: `/index.php?rest_route=/aibot/v1/chat`.
